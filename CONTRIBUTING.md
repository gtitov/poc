# Добавление и редактирование материалов

Изменения вносятся путем создания форка — копии репозитория в своём аккаунте, редактирования кода, а затем отправки обновлений обратно в оригинальный репозиторий через Pull Request (PR). Ниже будут приведены инструкции по работе с встроенной системой контроля версий в VS Code.

## Необходимое ПО

Для добавления материалов используется система контроля версий [Git](https://git-scm.com/) и платформа GitHub.

Для локального запуска сайта понадобится [NodeJS](https://nodejs.org/en) версии 24.

## Создание редактируемой копии

Пользователи не могут напрямую вносить изменения в репозиторий. Они создают собственную копию (*fork*), а затем предлагают "подтянуть" (*pull*) внесённые в собственную копию изменения владельцу основного репозитория. Это предложение "подтянуть" изменения называется *Pull Request* на GItHub или *Merge Request* в других сервисах.

1. **Сделать форк репозитория на GitHub**
   - Открыть страницу оригинального репозитория и нажать *Fork*, чтобы создать копию в своём аккаунте.

2. **Установка Git**
   - После установки Git на устройство необходимо указать метаданные пользователя, от имени которого в репозиторий будут вносится изменения. Без этих данных не получится сделать внести изменения. К обязательным метаданным пользователя относятся имя пользователя и электронная почта
   ```bash
   git config --global user.name "John Doe"
   git config --global user.email johndoe@example.com
   ```
   - Подробнее о Git https://git-scm.com/book/ru/v2/Введение-Первоначальная-настройка-Git

3. **Клонирование форка через VS Code**
   - Запустить VS Code.
   - Перейти в раздел *Source Control* (значок с веткой) или открыть командную палитру (Ctrl+Shift+P) и выбрать *Git: Clone*.
   - Вставить URL своего форка и выбрать папку для клонирования.
   - После клонирования VS Code предложит открыть проект.

4. **Установка зависимостей и запуск проекта**  
   - Открыть терминал в VS Code (Ctrl+`).
   - Выполнить команды:
     ```bash
     npm install
     npm run dev
     ```
   - Проект запустится, и локальный сайт будет доступен по адресу (в нашем случае - [http://localhost:4321](http://localhost:4321)).

5. **Редактирование кода и коммит изменений**
   - Внести необходимые правки в файлы проекта.
   - Перейти в раздел *Source Control*, где появится список изменённых файлов.
   - Нажать на знак «+» рядом с файлами или *Хранить все промежуточные изменения (Stage All Changes)*
   - Написать краткое и осмысленное пояснения для коммита. Описание коммита делаем на русском языке в формате «добавляет ...», например, «добавляет эссе Все карты лгут».
   - В разделе *Source Control* нажать *Commit* и *Publish* для отправки коммитов в свой форк.

6. **Создание Pull Request (PR)**
   - Перейти на сайт GitHub, открыть свой форк.
   - Нажать *Contribute*.
   - Добавить описание и отправить PR.
